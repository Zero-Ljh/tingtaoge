# 核心流程设计

> 版本：v0.1
> 状态：🟢 已定稿

---

## 1. 讨论流程

### 1.1 辩论模式

```
Round 1:
  Agent A（正方）: 发表立场，提出核心论据
  Agent B（反方）: 针对性反驳 A 的论据，提出对立观点
  Agent C（裁判）: 客观总结双方核心分歧，指出关键论点

Round 2:
  Agent A（正方）: 回应 B 的反驳，加固或修正立场
  Agent B（反方）: 进一步反驳，指出 A 的逻辑漏洞
  Agent C（裁判）: 评判本轮交锋，指出最有说服力的论据

Round 3:
  Agent A（正方）: 最终陈述，总结核心论点
  Agent B（反方）: 最终陈述，总结核心论点
  Agent C（裁判）: 最终裁决，列出双方有力论据和可能共识区
```

**规则**：
- 每人发言后必须直接回应前一个人的核心观点，不能自说自话
- 裁判在每轮结束时必须指出"本轮的亮点和薄弱点"

### 1.2 头脑风暴模式

```
Phase 1 - 发散（每人 1 轮）:
  Agent A（角色视角）: 从自身专业角度提出见解
  Agent B（角色视角）: 从互补视角补充
  Agent C（角色视角）: 从另一个维度扩展

Phase 2 - 交叉（每人 1-2 轮）:
  自由互相提问、补充、质疑
  不是"反驳"而是"再想想"

Phase 3 - 整合:
  每人提出一个整合性的建议
```

**规则**：
- 不强求反驳，鼓励补充和扩展
- 可以用"M 提到的 X 让我想到..."句式来连接

### 1.3 混合模式

```
Phase 1（头脑风暴阶段）: 各角色自由提出见解
  ↓ Agent C 的观点与 Agent A 明显对立 ↓
Phase 2（自动切换为辩论模式）: A 与 C 深入交锋，B 作为调和/裁判
  ↓ 交锋 2 轮后核心分歧已明晰 ↓
Phase 3（回到头脑风暴）: 在已有讨论基础上继续发散
  ↓ 再次检测到对立点 ↓
Phase 4（辩论模式）: 深入交锋
  ...循环直到结束
```

**触发规则**：
- 系统检测到 agent 发言中出现 "我不同意"、"恰恰相反"、"你的观点有问题" 等关键词时，自动转入辩论模式
- 如果连续 2 轮都没有新的对立观点，回到头脑风暴模式

## 2. 角色轮转规则

### 2.1 发言顺序

- **辩论模式**：固定 A → B → C → A → B → C ...
- **头脑风暴模式**：A → B → C → C → B → A（首轮正向，次轮反向，促进交叉）
- **混合模式**：动态按需决定

### 2.2 上下文传递

每轮发言时，传递给 agent 的上下文包括：
1. 自己的角色提示词（你是谁、什么立场）
2. 讨论原始论题
3. 截止目前完整的讨论记录
4. 上一个人的完整发言（强调要回应）
5. 如果是辩论模式，额外标注"请直接回应对手的上一轮观点"

---

## 3. 插话机制

### 3.1 触发方式

- 讨论流界面下方有输入框 + "插话"按钮
- 随时输入，点击发送

### 3.2 处理方式

```
用户发送插话
    → 插入到讨论记录中，标记为 "👤 你"
    → 下一个发言的 agent 收到的上下文包含：
      "[系统消息] 用户插话了：{用户消息}。请在发言中回应用户的参与。"
    → agent 在发言中直接回应
```

### 3.3 规则

- 插话不打断当前正在发言的 agent（等他说完）
- 插话插入后，下一个轮到的人必须回应
- 用户可连续插话（连续插话会合并处理）

---

## 4. 结束条件

### 4.1 自动结束

- 辩论模式：完成预设轮次（默认 3 轮）
- 头脑风暴模式：完成所有阶段（发散 + 交叉 + 整合）
- 混合模式：完成预设总轮数（默认 6 轮）

### 4.2 手动结束

- 用户点击"结束讨论"按钮
- 立即停止新发言，进入总结阶段

## 5. 结果整理

### 5.1 报告结构

```markdown
# 讨论报告：[论题]

## 基本信息
- 日期：YYYY-MM-DD
- 模式：辩论/头脑风暴/混合
- 参与角色：A / B / C

## 讨论摘要
（200 字以内概述）

## 各角色观点
### [角色A]
- 核心立场：
- 主要论据：
- 最终态度：

### [角色B]
...

### [角色C]
...

## 核心分歧
（列出了哪些点上有不同看法）

## 共识点
（有哪些点大家一致认可）

## 参考建议
（给用户作业的具体建议）
```

### 5.2 生成方式

- 讨论结束后，后端将所有讨论记录发给一个**总结专用的 Claude Code 调用**，prompt 要求按上述结构整理
- 结果保存到 `data/{timestamp}-report.md`
- 同时推送到前端展示
