# 产品需求文档 — 听涛阁

> 版本：v0.2
> 状态：🟢 已定稿

---

## 1. 背景与目标

### 1.1 背景

大学生在完成课程作业时经常需要向 AI 寻求意见，但当前 AI 助手**过于谄媚**——倾向于顺着用户观点说，很少主动提出质疑或补充对立视角。用户难以看到问题的全貌，做出真正客观的判断。

### 1.2 目标

构建一个多 agent 辩论与头脑风暴系统（**听涛阁**），让持有不同立场/视角的 AI agent 相互反驳、交叉验证、联想拓展，揭示问题的多面性。用户可以旁观也可以深度参与，最终获得结构化的输出结果。

### 1.3 核心理念

> **千涛拍岸，真相自览** — 不同观点如浪潮拍打，用户在听涛阁中静观波澜起伏，从中看清真相。

---

## 2. 用户角色

### 2.1 目标用户

- 高校学生（大一到大四），需要处理各种课程作业
- 需要多角度分析问题、寻找创意灵感的人

### 2.2 使用场景

| 场景 | 说明 | 推荐模式 |
|------|------|----------|
| 课程论文论证 | 抛出一个论点，看不同立场 agent 论证和反驳，完善论文论证结构 | 辩论 |
| 方案设计评估 | 提出方案，让不同角色从各自视角评估可行性 | 混合 |
| 创意头脑风暴 | 需要灵感时，让 agent 天马行空抛出创意 | 头脑风暴 |
| 渐进式深化 | 先发散想法 → 自动筛选 → 聚焦深化 | 头脑风暴 + 收束 |
| 观点辨析 | 对争议话题看正反方核心论据，形成自己的判断 | 辩论 |

---

## 3. 用户参与模型

听涛阁的核心特色：**用户不是旁观者，是第四位参与者。**

### 3.1 每轮用户发言

- 每轮 agent 辩论/讨论结束后，系统主动询问用户意见
- 输入框自动聚焦，提示"你怎么看？"
- 30 秒内不输入自动跳过
- 用户的发言进入历史记录，影响下一轮 agent 的上下文

### 3.2 随时插话

- 输入框**永远可用**，agent 说话时、暂停时均可打字
- 插话内容缓存，下个 agent 发言时必须直接回应
- 插话在 prompt 中标记为：
  > "!!! 这是用户的发言，你必须直接回应用户提出的观点，不得忽略 !!!"

### 3.3 阅读节奏控制

- 每个 agent 发言结束后出现 **「继续」按钮**
- 用户看完、想完后点击继续，才播放下一条
- 2 分钟不点击自动继续

## 4. 角色系统

### 4.1 预设角色库（8 个）

| 角色 | 代号 | 立场 | 标识色 |
|------|------|------|--------|
| 技术乐观派 | optimist | 相信进步能解决大部分问题 | 青 #00b4d8 |
| 怀疑论者 | skeptic | 保持批判态度，指出问题 | 橙 #f77f00 |
| 伦理学者 | ethicist | 关注伦理与社会影响 | 紫 #7b2cbf |
| 实用主义者 | pragmatist | 关注实际可行性 | 绿 #2d6a4f |
| 理想主义者 | idealist | 坚持最高标准 | 金 #e0a800 |
| 现实主义者 | realist | 基于事实和现状 | 棕 #9c6644 |
| 跨界创新者 | innovator | 打破思维定式 | 粉 #e3646b |
| 数据分析师 | analyst | 用数据和逻辑说话 | 蓝 #457b9d |

### 4.2 各模式默认角色

| 模式 | 默认角色组合 |
|------|-------------|
| 辩论 | 乐观派 vs 怀疑论者，伦理学者裁判 |
| 头脑风暴 | 创新者 + 实用主义者 + 伦理学者 |
| 混合 | 乐观派 + 怀疑论者 + 现实主义者 |

## 5. 三种讨论模式

### 5.1 辩论模式

**流程**：严格回合制，立场对立交锋

```
第1轮  正方立论 → 反方反驳 → 裁判点评
        ↓     （交换发言顺序）
第2轮  反方先攻 → 正方还击 → 裁判点评
        ↓
第3轮  正方总结 → 反方总结 → 裁判裁决
```

**规则**：
- 每人必须直接回应对手的核心观点
- 偶数轮交换先手顺序，防止一方永远被"后手反驳"
- 裁判每轮指出"本轮的亮点和薄弱点"

### 5.2 头脑风暴模式

**两种子模式**：

**自由发散**（默认）：
- agent 不需要回应或附和前一个人
- 可以提出疯狂、异想天开的想法
- 数量比质量重要
- 具体点子比抽象理论更有价值

**逐轮收束**（可选）：
```
第1轮  发散 → agent 天马行空抛想法
        ↓  Claude 自动筛选总结
第2轮  聚焦 → 在最有潜力的 1-2 个方向上深入
        ↓  Claude 再次筛选
第3轮  深化 → 给出具体方案/建议
```

### 5.3 混合模式

- 初始像头脑风暴自由发散
- 系统检测到对立观点关键词时自动切入辩论模式
- 交锋 2 轮后回到发散状态
- 关键词：我不同意、恰恰相反、不敢苟同、你的观点有问题、我反对等

## 7. 用户故事

- **课程论文场景**：我输入论文论点"人工智能是否会取代程序员"，选择辩论模式。乐观派和怀疑论者激烈交锋，我从中学到了双方的核心理由，完善了论文的论证结构。

- **头脑风暴场景**：我有个作业需要创意方案，选头脑风暴模式+逐轮收束。第一轮 agent 天马行空抛出 10 多个想法，系统自动筛选出两个方向，第二轮聚焦深入，我得到了可落地的方案。

- **插话场景**：agent 们讨论时忽略了一个角度（比如教育成本），我直接在输入框打字插话。下个 agent 收到"用户插话"的强制回应指令，认真回应了我的观点。

- **导出场景**：讨论结束后，我点击"总结 HTML"下载了一个带暗色主题的 HTML 文件，直接整理进作业提交。

| 要求 | 指标 |
|------|------|
| 实时性 | SSE 推送，单轮响应不超过 2 分钟 |
