# 听涛阁 — 需求全景梳理

> 整理时间：2026-05-11
> 对应版本：PRD v0.2

---

## 一、用户画像与情景

### 你是谁
- 大一学生，有各种课程作业
- 需要 AI 帮助，但现有的 AI 太顺着你说（谄媚），看不到真实的多面性
- 懂一些 Python，用 Windows 系统

### 你为什么要用听涛阁
- 写课程作业时想听不同角度的意见
- 不只是要答案，更想看到观点碰撞的过程
- 想要一个能反驳你、挑战你、帮你把问题想透的工具
- 作业场景举例：思政小论文、方案设计、观点分析、创意头脑风暴

### 你希望怎么用
1. 打开浏览器，输入论题，选模式
2. 旁观 3 个 agent 互相辩论/头脑风暴
3. 随时参与进去，说自己的想法
4. 看完讨论过程，导出报告整理进作业

## 三、三种模式详细对比

| 维度 | 辩论 | 头脑风暴 | 混合 |
|------|------|----------|------|
| **核心目的** | 辨明是非，找出最优论点 | 打开思路，激发灵感 | 先发散再聚焦 |
| **Agent 关系** | 对立 + 裁判 | 互补，各说各的 | 先互补，检测到冲突后自动交锋 |
| **发言规则** | 必须回应对手观点 | 不需要回应前一个人，尽管天马行空 | 自然切换 |
| **轮换顺序** | A→B→C (偶数轮换先手) | A→B→C→C→B→A (正反交替) | 动态按需 |
| **结束方式** | 3 轮后裁判总结 | 发散→交叉→整合 | 自然收敛 |
| **适合场景** | 论文论证、方案评估 | 创意作业、灵感枯竭 | 复杂问题需要多角度+深挖 |

## 五、用户交互流程

```
用户打开浏览器
    ↓
输入论题，选择模式，点击开始
    ↓
第1轮 agent 辩论/头脑风暴
    ↓
系统问：「你怎么看？」 ← 用户发言位（30秒超时）
    ↓
第2轮 agent 带着用户观点继续
    ↓
系统再问：「现在你怎么看？」
    ↓
...（循环）
    ↓
讨论结束 → 自动生成报告
    ↓
用户选择：导出总结(MD/HTML) / 导出原始记录(MD/HTML)
```

**任何时候**：用户可以直接在底部输入框插话，下个 agent 强制回应。

## 六、技术架构

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | 原生 HTML/CSS/JS | 单页应用，零构建，零依赖 |
| 后端 | Python FastAPI + uvicorn | HTTP + SSE 服务 |
| Agent 引擎 | Claude Code CLI | 通过 subprocess 调用 `claude -p` |
| 数据存储 | 本地文件系统 | `data/{id}-report.md` / `data/{id}-transcript.md` |
| 通信协议 | SSE (Server-Sent Events) | 实时推送讨论流 |

```
浏览器 ←SSE→ FastAPI ←subprocess→ Claude Code CLI
                    │
                    ├─ Agent 调度器（管理 3 个 agent 轮流发言）
                    ├─ 角色系统（人格 + 立场 + prompt 构建）
                    └─ 讨论记录（原始对话 + 总结报告）
```

## 七、设计理念

1. **用户即参与者** — 你不是旁观者，你和 agent 是平等的讨论者
2. **碰撞出真知** — 不同观点的碰撞才有深度，不是单方面灌输
3. **节奏由你控** — 每条消息后可暂停，看完再继续
4. **先发散后收敛** — 头脑风暴先天马行空，再筛选聚焦
5. **本地离线** — 不依赖云 API，用你自己的 Claude Code 跑
6. **数据是你的** — 所有记录在本地，可以导出带走
