## 2. 模块划分

### 2.1 Web 前端 (`static/`)
- **职责**：提供用户界面，通过 HTTP/SSE 与后端通信
- **核心文件**：`index.html`（内嵌 CSS + JS）
- **关键能力**：SSE 流式渲染、插话输入、报告展示

### 2.2 FastAPI 后端 (`backend/`)
- **职责**：HTTP 服务、SSE 推送、Agent 调度、Claude CLI 通信
- **核心文件**：
  - `main.py` — 应用入口，路由注册
  - `agent_manager.py` — Agent 调度器（角色分配、轮转逻辑）
  - `roles.py` — 角色人格库（预设 + 自定义）
  - `claude_client.py` — Claude Code CLI 子进程通信

### 2.3 Agent 调度器
- **职责**：决定当前轮到谁发言、传递上下文、注入插话
- **轮转规则**：
  - 辩论模式：严格 A→B→C→A 循环
  - 头脑风暴模式：自由轮转，可指定下一个发言人
  - 混合模式：按规则自动切换

### 2.4 Claude CLI 通信模块
- **职责**：通过 `subprocess` 启动 Claude Code CLI，传入格式化 prompt
- **通信协议**：
  - 输入：标准化的角色 prompt + 当前讨论上下文 + 前一个人发言
  - 输出：读取 stdout，流式返回

## 3. 数据流

### 3.1 正常讨论流程

```
用户提交论题
    → POST /api/start {topic, mode, roles}
    → 后端初始化 3 个 agent 的状态
    → Agent 调度器 决定第一个发言人
    → 后端通过 subprocess 调 Claude CLI:
        "你是[角色A]，请用[立场]对论题[XXX]发表观点"
    → 读取 stdout，通过 SSE 流式推送到浏览器
    → 轮到 Agent B，同样的流程，但上下文包含 A 的发言
    → 循环直到预定轮次或用户结束
    → 生成总结报告
```

### 3.2 用户插话流

```
用户点击插话按钮，提交文字
    → POST /api/interject {message}
    → Agent 调度器将插话注入为下一轮上下文
    → 当前轮到/下一个 agent 需要回应用户插话
    → SSE 推送 "用户插话" 事件到前端
    → 正常流程继续
```

### 3.3 异常处理

- **CLI 超时**：当前 agent 发言超 60s 则跳过，轮到下一个
- **CLI 崩溃**：记录错误，尝试重启，通知用户
- **所有 agent 失败**：返回错误信息到前端
